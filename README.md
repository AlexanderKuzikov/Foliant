# Foliant

Фоновая утилита для автоматической обработки папок с PDF-задачами — растрирует страницы и собирает их в готовый к печати документ с автоматическим выбором макета.

Поддерживаемые платформы: **Windows 10/11**, **Linux**.

## Как это работает

Foliant сканирует рабочую директорию на наличие вложенных папок. Каждая папка — это отдельная задача, содержащая один или несколько PDF-файлов. При запуске утилита обрабатывает все задачи последовательно.

Файлы с именем, начинающимся на указанный префикс (`outputFilePrefix`), пропускаются — это позволяет безопасно запускать утилиту повторно без переобработки уже готовых документов.

### Логика выбора макета

Макет выбирается автоматически на основе суммарного количества страниц всех PDF в задаче:


| Кол-во страниц | Макет                                 | Ориентация листа | DPI растрировки |
| -------------- | ------------------------------------- | ---------------- | --------------- |
| ≤ 19           | 1 страница на лист                    | Книжная          | 300 DPI         |
| 20 – 38        | 2 страницы на лист (повёрнуто на 90°) | Книжная          | 200 DPI         |
| > 38           | 4 страницы на лист (2×2)              | Книжная          | 150 DPI         |


Все итоговые файлы имеют **книжную ориентацию A4** — формат, принимаемый сервисом электронных отправлений Почты России.

Страницы вписываются в ячейки без обрезки с сохранением пропорций. Количество листов в итоговом файле не ограничено.

### Режим 2 страницы на лист

Две исходные страницы размещаются друг под другом на книжном листе A4, каждая повёрнута на 90°. Это позволяет максимально задействовать площадь ячейки вне зависимости от ориентации исходных страниц.

### Структура рабочей директории

```
C:/tasks/
├── order_001/
│   ├── document.pdf
├── order_002/
│   ├── part1.pdf
│   ├── part2.pdf
└── logs/
│   └── run_2026-03-17T10-35-00.log
└── fol__order_001.pdf   ← результат
└── fol__order_002.pdf   ← результат
```

## Требования

- [Node.js](https://nodejs.org/) версии 18 и выше
- Windows 10 / 11 или Linux

## Установка

```bash
# Клонировать или скачать репозиторий
git clone https://github.com/AlexanderKuzikov/Foliant.git
cd Foliant

# Установить зависимости
npm install

# Собрать проект
npm run build
```

## Настройка

Перед первым запуском отредактируйте `config.json`:

```json
{
  "workDir": "C:/tasks",
  "logDir": "C:/tasks/logs",
  "outputFilePrefix": "fol_",
  "deleteSourcePdfs": false,
  "maxPagesPerDocument": 19,
  "margins": {
    "top": 15,
    "right": 10,
    "bottom": 15,
    "left": 10
  },
  "layout": {
    "single": {
      "pagesPerSheet": 1,
      "cols": 1,
      "rows": 1,
      "orientation": "portrait",
      "rasterDpi": 300,
      "jpegQuality": 75
    },
    "twoPerSheet": {
      "pagesPerSheet": 2,
      "cols": 1,
      "rows": 2,
      "orientation": "portrait",
      "rotateContent": 90,
      "rasterDpi": 200,
      "jpegQuality": 75
    },
    "fourPerSheet": {
      "pagesPerSheet": 4,
      "cols": 2,
      "rows": 2,
      "orientation": "portrait",
      "rasterDpi": 150,
      "jpegQuality": 75
    }
  }
}
```


| Параметр                 | Описание                                                                       |
| ------------------------ | ------------------------------------------------------------------------------ |
| `workDir`                | Рабочая директория с папками задач                                             |
| `logDir`                 | Директория для сохранения логов                                                |
| `outputFilePrefix`       | Префикс имени выходного файла (пустая строка — без префикса)                   |
| `deleteSourcePdfs`       | Удалять исходные PDF после обработки                                           |
| `maxPagesPerDocument`    | Порог страниц для переключения на следующий макет                              |
| `margins`                | Поля страницы в миллиметрах (top, right, bottom, left)                         |
| `layout.*.orientation`   | Ориентация итоговой страницы (`portrait` / `landscape`)                        |
| `layout.*.rotateContent` | Поворот каждого изображения в ячейке в градусах (0, 90, 180, 270). Опционально |
| `layout.*.rasterDpi`     | DPI растрировки для каждого макета                                             |
| `layout.*.jpegQuality`   | Качество JPEG для каждого макета (1–100)                                       |


Пути поддерживают как прямой слэш (`C:/tasks`), так и обратный (`C:\tasks`).

## Использование

### Windows

Двойной клик по `run.vbs`. Окно терминала не появляется. В системном трее появится иконка с текущим статусом обработки. По завершении всех задач иконка автоматически исчезает через 10 секунд.

Меню иконки в трее:

- Текущий статус обработки
- Открыть папку задач
- Открыть логи
- Выход

### Linux

Утилита запускается из внешнего окружения и завершается автоматически. Код возврата сигнализирует о результате:

```bash
node dist/index.js

echo $?   # 0 — все задачи выполнены успешно
          # 1 — одна или несколько задач завершились с ошибкой
```

### Для разработчика

```bash
# Режим разработки (без предварительной сборки)
npm run dev

# Продакшн
npm run build
npm start
```

## Стек технологий

- **[pdf-lib](https://pdf-lib.js.org/)** — сборка PDF и компоновка страниц
- **[@hyzyla/pdfium](https://github.com/hyzyla/pdfium)** — растрировка через Google PDFium (WebAssembly)
- **[sharp](https://sharp.pixelplumbing.com/)** — конвертация растровых изображений в JPEG
- **[systray2](https://github.com/felixneufeld/systray2)** — иконка в системном трее (только Windows)
- **[winston](https://github.com/winstonjs/winston)** — логирование
